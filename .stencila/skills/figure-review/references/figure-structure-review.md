---
title: Figure Structure Review
description: Condensed review checklist for Stencila figure fences, captions, subfigures, layouts, and cross-references.
---

## Figure fences and caption separation

Use `::: figure` fences for figures.

```smd
::: figure

![](image.png)

Caption text.

:::
```

Review checks:

- Content paragraphs containing only an image, audio, or video become figure content.
- Other paragraphs become caption content.
- Flag cases where prose intended as caption is accidentally mixed into figure content or vice versa.

## Subfigures

Nested subfigures must be indented by four spaces inside the parent figure:

```smd
::: figure [2]

    ::: figure

    ![](panel-a.png)

    Panel A caption.

    :::

    ::: figure

    ![](panel-b.png)

    Panel B caption.

    :::

Overall caption.

:::
```

Review checks:

- Missing indentation means the blocks are sibling figures, not subfigures.
- Panel order in the source should match how the caption discusses the panels.
- Automatic subfigure labels usually make manual A/B/C overlay badges unnecessary.

## Layout mini-language

Common patterns:

| Pattern | Meaning |
|---|---|
| `[row]` | Single row |
| `[2]` / `[3]` | Equal-width columns |
| `[30 70]` | Proportional columns |
| `[40 g20 40]` | Columns with an explicit gap |
| `[a b \| a c]` | Layout map with spanning panel |
| `[a a \| b c]` | Top panel spans two columns |
| `[a . \| b c]` | Empty cell |
| `[30 70 : a b \| a c]` | Widths plus layout map |

Review checks:

- The number and order of subfigures should match the layout map.
- Repeated layout letters should correspond to intentional spanning behavior.
- Flag layouts that obscure the intended panel comparison or make the caption difficult to follow.

## Cross-references

Review checks:

- Figure references should target the intended figure.
- Auto-numbering and autogenerated subfigure labels should be preferred over hard-coded numbering where possible.
- Flag nearby text that refers to the wrong panel or figure number after a figure reorder.
