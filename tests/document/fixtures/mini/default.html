Mini

Stencila has it's own language for reproducible, data-drive, research called Mini. An important aspect of reproducibility is readability. Mini aims to be as easy to read and understand as it is to write. Mini is meant to be only slightly more advanced that the expressions that your write in your calculator or into the cell of a spreadsheet

Using other languages

An important feature of Mini is it's ability to interoperate with other languages. Mini can act as a glue language - for those times when you need to combine the strengths of several languages. For example, you might want to extract some data from a database using Structured Query Language (SQL), then perform a specialised statistical analysis on the data using R, then visualise the results using Mini's built in plotting functions.

Mini makes this possible through alternative execution contexts. Each context knows how to convert values to and from the Mini value types and how to execute snippets of code in the context's native language. Currently, only two execution contexts are supported: the JsContext (for executing Javascript code in a browser or desktop window) and the NodeContext (for executing Javascript code within Node.js). The RContext and PythonContexts are planned for the 0.26 release with more language contexts following soon.

There are two special functions in Mini for executing code in other languages: call and run.

The call function

The call function executes code within a function scope. That means that any variables that you declare within that function are local to the function and won't leak into the context's global scope. So, when you type call() into a cell, your effectively saying to Mini take this code, make it into a function, execute the function, and give me back the value. Here's an example of a simple call to some Javascript,

let names = ['Hiphop', 'opotamus', ' v ', 'Rhyme', 'noceros']
return names.join('')
Hiphopopotamus v Rhymenoceros

Notice that because we ended the JavaScript code, with a return statement that the cell has an output. When you call() some JavaScript code it's like executing (function(){code})(). The same is true for other external languages, it's just that the syntax differs.

Just like with other Mini cells, you can assign the output of a call() to a document variable,

return Math.PI
3.141592653589793

A call to another language is not very useful unless you provide it with some data. This is where Mini's glueing comes in. Let's create an array in Mini,

apple,pear,kiwi

and then use Javascript to join it together,

return 'My favourite fruits: ' + fruits.map((fruit,index) => {
return `${index + 1}. ${fruit}`
}).join(', ')
My favourite fruits: 1. apple, 2. pear, 3. kiwi

The nice thing about this is that Mini's dependency analysis still works, so you can change the fruits array and the output above will change - try it!

Notice that our Javascript function got supplied an argument called fruits - which is the same array of strings as in our documentent variable called fruits. You can also provide named arguments to call,

return my_favourite_fruits[index]
kiwi
